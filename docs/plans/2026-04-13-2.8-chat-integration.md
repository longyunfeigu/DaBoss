## §1 关键设计决策

### (A) v2 persona 被聊天服务"看见"

现状：`get_persona_loader()` 每次请求 new PersonaLoader，不会 `refresh_from_db`。所以 v2 DB 里的 persona 被聊天服务完全看不见。**修复**：在 `get_stakeholder_chat_service` 和其他消费 persona_loader 的依赖里，先 `await loader.refresh_from_db(repo)`，再返回服务。**风险**：每个请求多一次 DB query。**缓解**：PersonaLoader 本身有 30s TTL 缓存；refresh_from_db 只在 TTL 过期时触发真正的 DB 查询（需要在 loader 里加最后刷新时间）。**MVP 简化**：本 Story 直接在依赖里每次 refresh，性能代价 1 次 `SELECT ... WHERE schema_version=2`，可接受；未来 Story 优化。

### (B) prompt_builder v2 扩展

新增函数 `build_system_prompt_v2(persona: Persona) -> str`，输出结构：

```
你是「{name}」...

## Hard Rules（底线，绝不退让）
- {statement} [severity: {severity}]
- ...

## Identity（身份背景）
背景：{background}
核心价值：{core_values}

## Expression（表达风格）
语气：{tone}
常用口头禅：
- {catchphrase}
打断倾向：{interruption_tendency}

## Decision（决策模式）
...

## Interpersonal（人际互动）
...

## 对抗化字段（不轻易暴露）
【隐藏议程 — 不要在对话中直接说出来，但所有决策都围绕它】
{hidden_agenda}

（后接标准的 _ROLE_BEHAVIOR_INSTRUCTION + 场景 + 情绪标注）
```

对抗化字段来自 `identity.hidden_agenda` + `hard_rules[severity=critical]`。

### (C) 分流点

`_generate_reply` 里：
```python
if persona.schema_version >= 2:
    system_prompt = build_system_prompt_v2(persona, org_context=..., scenario_context=...)
    # 构造 messages 同 v1 path
else:
    # 走旧的 build_compressed_llm_messages
```

为了最小改动，**新增 `build_compressed_llm_messages_v2`** 复用三区结构但用 v2 system prompt 替换 persona_content 块。

### (D) create_room_from_persona

新方法放在 `BattlePrepService`（符合 AC 字面），但也可独立。决定放 `BattlePrepService` 扩展。

```python
async def create_room_from_persona(self, persona_id: str, user_id: str = "anonymous") -> ChatRoomDTO:
    """For Story 2.8 编辑器入口 — 创建 chatroom 指向已有 persona。"""
    persona = self._persona_loader.get_persona(persona_id)
    if persona is None:
        raise ValueError(f"persona {persona_id} not found")
    room = await self._chatroom_service.create_room(
        CreateChatRoomDTO(
            name=f"演练: {persona.name}",
            type="private",
            persona_ids=[persona_id],
        )
    )
    return room
```

### (E) 前端 wiring

替换 Story 2.7 的 `handleStartBattle` alert：调 `POST /api/v1/stakeholder/personas/{id}/start-battle` → `navigate(/chat/{roomId})`。"有未保存改动时点演练"仍弹 ConfirmDialog（保存或继续）。

### (F) 集成测试策略

AC6 要求 "e2e → AI 首轮回复文本匹配 persona.expression.catchphrases 中至少 1 条"。真正跑 LLM 贵、脆弱。**采用 fake LLM**：mock 一个 LLM 让它echo system prompt 的前 200 字，断言其中包含催话术。这测的是 prompt 注入正确性，不测 LLM 真实行为——符合单元测试边界。

```ts
export async function startBattleFromPersona(id: string): Promise<{ id: number }> {
  const resp = await fetch(`${API_BASE}/personas/${id}/start-battle`, { method: 'POST' })
  if (!resp.ok) {
    const err = await resp.json().catch(() => null)
    throw new Error(err?.error?.message || `Failed to start battle: ${resp.status}`)
  }
  const json: ApiResp<{ id: number }> = await resp.json()
  return { id: json.data.id }
}
```
